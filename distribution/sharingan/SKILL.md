# sharingan

Mac の音声をリアルタイムに日本語文字起こしし、その会話ログをこの AI が読んで質疑応答するためのツール。話者は `自分`(マイク)と `相手`(システム音声)の2種類。文字起こしは `[時刻] 話者: 本文` の形式でファイルへ追記され続ける。

## 前提

- macOS 26 以降。`sharingan` と `sharingan-session` が PATH 上にあること(未導入ならリポジトリの `install.sh` を実行して導入する)。
- 初回の録音開始時に、マイク・音声認識の許可ダイアログが出る。利用者に許可してもらう。
- 初回は日本語(ja-JP)の認識モデルのダウンロードが走ることがある。

## 録音の制御

背景で録音セッションを動かす。すべて `sharingan-session` 経由で行う。

録音を開始する:

```sh
sharingan-session start
```

録音を停止する(末尾の発話を確定してから終了する):

```sh
sharingan-session stop
```

状態を確認する(録音中か、どのファイルに書いているか):

```sh
sharingan-session status
```

最新の文字起こしファイルのパスを得る:

```sh
sharingan-session latest
```

## 会話について質疑応答する

利用者が会話の内容(例:「相手がさっき言った見積もりの話は?」)を尋ねたら:

1. `sharingan-session latest` で現在の文字起こしファイルのパスを得る。
2. そのファイルを読む。長い場合は全体ではなく末尾(直近の発話)を優先して読む。
3. `自分` と `相手` のラベルを踏まえて、会話の文脈に沿って答える。

録音中でもファイルは随時追記・フラッシュされるので、その時点までの内容を読める。

## 既知の挙動・制約

- **書き出しは確定した発話のみ**。喋っている途中の暫定結果はファイルに出ない(数秒遅れて確定行が追記される)。
- **自分/相手の区別は音声の経路による**。マイク=自分、システム音声=相手。同じ経路に複数人がいても、その中の誰かまでは区別しない。
- **システム音声(相手)は安定した署名がないと無音で失敗する**。相手側がまったく文字起こしされない場合は、`install.sh` による署名込みの導入ができているかを確認する。マイク(自分)は署名なしでも動く。
- 言語は日本語(ja-JP)固定。文中の英語技術用語は拾えるが、精度は完全ではない。
