# sharingan

Mac の音声をリアルタイムに日本語文字起こし・録音し、その会話ログをこの AI が読んで質疑応答するためのツール。実体はメニューバー常駐アプリ(sharingan.app)で、CLI `sharingan` はアプリへ命令を送る窓口。話者は `自分`(マイク)と `相手`(システム音声)の2種類。文字起こしは `[時刻] 話者: 本文` の形式でファイルへ追記され続ける。

## 前提

- macOS 26 以降。`sharingan` CLI が PATH 上にあり、sharingan.app が導入済みであること(未導入ならリポジトリの `install.sh` を実行して導入する)。
- 初回のセッション開始時に、マイク・音声認識の許可ダイアログが出る。利用者に許可してもらう。
- 初回は日本語(ja-JP)の認識モデルのダウンロードが走ることがある。

## セッションの制御

セッションを開始する(アプリが未起動なら自動で起動する。録音と文字起こしの両方がオンで始まる):

```sh
sharingan start
```

録音せず文字起こしだけ、のように片方を切って始めることもできる:

```sh
sharingan start --no-record       # 文字起こしのみ
sharingan start --no-transcribe   # 録音のみ
```

セッション途中で片方だけ切り替える:

```sh
sharingan set record off      # 録音だけ止める(文字起こしは続く)
sharingan set transcribe on   # 文字起こしを再開する
```

セッションを停止する(末尾の発話を確定し、録音ファイルを閉じてから終了する):

```sh
sharingan stop
```

状態を確認する(セッション中か、録音/文字起こしのオンオフ、どのファイルに書いているか):

```sh
sharingan status
```

最新の文字起こしファイルのパスを得る(アプリ未起動でも答えられる):

```sh
sharingan latest
```

## 会話について質疑応答する

利用者が会話の内容(例:「相手がさっき言った見積もりの話は?」)を尋ねたら:

1. `sharingan latest` で現在の文字起こしファイルのパスを得る。
2. そのファイルを読む。長い場合は全体ではなく末尾(直近の発話)を優先して読む。
3. `自分` と `相手` のラベルを踏まえて、会話の文脈に沿って答える。

セッション中でもファイルは随時追記・フラッシュされるので、その時点までの内容を読める。

## 保存場所

セッションごとに `~/Library/Application Support/sharingan/sessions/<日時>/` へまとまる:

- `transcript.md`: 文字起こし
- `audio.m4a`: 録音(ステレオ1ファイル。左=自分、右=相手)。録音オフのセッションには無い
- `summary.md`: アプリで生成した要約(あれば)

過去セッションの閲覧・再生・削除・要約はアプリ(メニューバーの目のアイコン → 履歴を開く)でできる。

## 既知の挙動・制約

- **書き出しは確定した発話のみ**。喋っている途中の暫定テキストはファイルに出ない(数秒遅れて確定行が追記される)。暫定テキストはアプリのライブ表示でだけ見える。
- **自分/相手の区別は音声の経路による**。マイク=自分、システム音声=相手。同じ経路に複数人がいても、その中の誰かまでは区別しない。
- **システム音声(相手)は安定した署名がないと無音で失敗する**。相手側がまったく文字起こしされない場合は、`install.sh` による署名込みの導入ができているかを確認する。`sharingan status` の注意行にも出る。
- 言語は日本語(ja-JP)固定。文中の英語技術用語は拾えるが、精度は完全ではない。
- 録音を途中でオフ→オンすると、オフの間の音声はファイルに入らない(再生時間がその分短くなる)。
