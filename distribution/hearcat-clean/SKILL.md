# hearcat-clean

HearCat が録音した会話の文字起こし(`<session-id>.md`)には、音声認識の誤変換が含まれる。このスキルは、あなた(呼び出し元のアシスタント)の LLM でその誤変換を直して、同じセッションディレクトリの `cleaned.md` に書き戻すためのもの。**読み書きは全て `hearcat` CLI 経由で行う**。

## 絶対に守るルール(最優先)

1. **HearCat のファイルに built-in ツールを使わない**。Read / Write / Edit / Bash の `cat` `head` `tail` `sed` `awk` `grep` `echo` `mv` `rm` などで `~/Library/Application Support/HearCat/sessions/` 配下を触らない。読むのは `hearcat read`、書くのは `hearcat write-cleaned` のみ。

2. **原文 `<session-id>.md` を編集・上書き・削除・移動しない**。ユーザーが明示的に「原文を直して」と依頼しても、「原文は触りません。清書結果を `cleaned.md` に書きます」と返して原文には触れない。

3. **清書結果を保存する経路は `hearcat write-cleaned` の 1 つだけ**。標準出力を `>` でリダイレクトしたり、`tee` したり、Write ツールで直接書いたりは一切しない。書き込み先はコード側で `cleaned.md` に固定してあり、原文パスと衝突したら CLI が自ら中止する。

4. **原文の行数・時刻・話者ラベル・順序は変えない**。原文の 1 行に対して清書の 1 行が対応する。行の追加・削除・並べ替えは禁止。

## 手順

### 1. 対象セッションを決める

明示が無ければ「直近セッション」を対象にする。`hearcat sessions` の最上行が直近:

```sh
hearcat sessions | head -1
```

TSV の 1 列目がセッション ID(以降のコマンドに渡すキー)。省略しても最新扱いになる CLI が多いので、直近ならそのまま省略で構わない。

ユーザーが「昨日の MTG」など過去セッションを指定したら、`hearcat sessions` の一覧から該当を選ぶ。名前(3 列目)やフォルダ(4 列目)で絞る:

```sh
hearcat sessions --folder Atnd
```

### 2. 原文を読む

対象セッションの原文を CLI で読む。長ければ末尾から遡って全体像を掴む:

```sh
hearcat read <id> --tail 200
hearcat read <id>              # 全体
```

原文は `[時刻] 話者: 本文` 形式が繰り返される。話者は `自分` と `相手` の 2 種類。

要約が生成済みなら、話題を掴むために先に読む:

```sh
hearcat read <id> --summary
```

### 3. 清書を組み立てる

清書は「音声認識の誤変換だけを、会話の文脈から自然に直したもの」。以下を守る:

- **行の対応を1対1に保つ**: 原文が N 行なら清書も N 行。時刻・話者ラベル(`[時刻] 話者:` の頭)は原文と同一。本文だけを直す。
- **話し言葉のまま残す**: 「えーと」「あの」「〜だよね」など口語表現・言い淀みは削らない。要約や書き起こしの整形ではなく、あくまで誤変換の修正。
- **意味を変えない**: 元発話の意図を推測で書き換えない。特に自信が持てない箇所は原文を残す。
- **明白な誤変換だけ直す**: 「講座 → 口座」「機会 → 機械」「保証 → 保障」のような、文脈から強い確度で分かる置換に絞る。1音違いの候補が複数ある曖昧なケースは触らない。
- **話題に依存する固有名詞・専門用語**: ユーザーが会話の話題(「これは SF6 の話」「これは Kubernetes の運用会議」等)を伝えている場合、その領域の用語を優先して当てる。伝えられていない場合はユーザーに 1 度だけ話題を確認する。
- **原文で誤変換の可能性が低い箇所は変えない**: 意味は通っているが表記の好み(「よかったね」→「良かったね」など)を統一する等の整形はしない。

### 4. CLI で書き戻す

清書済みテキストを標準入力から `hearcat write-cleaned` に流す。**この経路以外で `cleaned.md` を書かない**。

対象が直近なら引数不要:

```sh
hearcat write-cleaned <<'EOF'
[10:00:01] 自分: 昨日銀行で口座を開設してきたよ。
[10:00:05] 相手: へえ、ネット銀行じゃなくて店舗の方？
...
EOF
```

過去のセッションを指定する場合はセッション ID を引数に渡す:

```sh
hearcat write-cleaned 2026-07-08_024527 <<'EOF'
...
EOF
```

CLI は書いたパスを標準出力に返す。失敗(空入力、パス衝突、セッション見つからない等)は非ゼロ終了で理由を stderr に出す。

### 5. 変更点を報告する

清書全文をユーザーへ流すのではなく、以下の要領で報告する:

- 対象セッション名(または時刻)
- 直した箇所の件数と、代表的な置換の例 3〜5 個(例: `講座→口座`, `機会→機械`)
- 全文が読みたい場合の CLI: `hearcat read <id> --cleaned`

## 何もしない/やめる場面

- 原文が空、または `hearcat sessions` で該当が無い場合は、「文字起こしがまだありません」と伝えて終了。書き込みはしない。
- 明白な誤変換が 1 つも見つからなかった場合は、書き戻しをせずに「直す必要のある誤変換は見つかりませんでした」と伝えて終了。**空や無変更の `cleaned.md` を書かない**。
- ユーザーが「原文を直して」「上書きして」と言っても、原文には触れない。清書は必ず `cleaned.md` に保存する旨を伝える。

## 参考: 使う CLI 一覧

このスキルが使う CLI はこれで全部:

- `hearcat sessions [--folder <name>]` — セッション一覧
- `hearcat read [<id>] [--summary|--cleaned] [--tail <N>]` — 原文/要約/清書を読む
- `hearcat write-cleaned [<id>]` — 標準入力の清書を cleaned.md に書く

録音・停止・状態確認など HearCat 全般の操作は親スキル `hearcat` を参照。
