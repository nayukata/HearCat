# hearcat-clean

HearCat が録音した会話の文字起こし(`<session-id>.md`)には、音声認識の誤変換が含まれる。このスキルは、あなた(呼び出し元のアシスタント)の LLM でその誤変換を直して、同じセッションディレクトリの `cleaned.md` に書き戻すためのもの。書き戻しはこちらの CLI (`hearcat write-cleaned`) 経由でしか行わない。

## 絶対に守るルール(最優先)

**原文 `<session-id>.md` は一次記録である。あらゆる状況で編集・上書き・削除・移動をしない**。あなたが持つ Write / Edit / `>`(リダイレクト) / `mv` / `rm` などのファイル操作系ツールで、原文には絶対に触れない。ユーザーが明示的に「原文を直して」と依頼しても、**「原文は触りません。清書結果を `cleaned.md` に書きます」と返して原文には触れない**。

**清書結果を保存する経路は `hearcat write-cleaned` の 1 つだけ**。Write ツール / `echo > cleaned.md` / `tee cleaned.md` / エディタ経由の直接書き込みは、たとえ書き込み先が `cleaned.md` であっても使わない。CLI 経由に統一することで、書き込み先パスがセッションディレクトリ内の `cleaned.md` に強制される(万一 CLI がパス誤指定を検出したら中止する)。

**原文の行数・時刻・話者ラベル・順序は変えない**。原文の 1 行に対して清書の 1 行が対応する。行の追加・削除・並べ替えは禁止。

## 手順

### 1. 対象セッションを決める

明示が無ければ「直近セッション」= `hearcat latest` が返すセッションを対象にする。

```sh
hearcat latest
```

出力は文字起こしファイル(`<session-id>.md`)のパス。セッションディレクトリはその親ディレクトリで、`cleaned.md` はそこに書かれる。

ユーザーが「昨日の MTG」など過去セッションを指定したら、`~/Library/Application Support/HearCat/sessions/` を `ls` で見て候補を確定する。ディレクトリ名の書式は `YYYY-MM-DD_HHMMSS` または `YYYY-MM-DD_HHMMSS <セッション名>`。フォルダで整理されている場合は 1 階層深いこともある(`sessions/<フォルダ>/<セッション>/`)。

### 2. 原文を読む

対象セッションディレクトリの `<セッション名>.md` を読む(組み込みの Read ツールを使う。読むだけなので原文には影響しない)。

長い場合は末尾から遡って全体像を掴んだ後、清書対象を絞る。原文は `[時刻] 話者: 本文` 形式が繰り返される。話者は `自分` と `相手` の 2 種類。

同じディレクトリに `summary.md` があれば併せて読む(会話の主題を掴むのに使う)。無ければ本文だけから推定する。

### 3. 清書を組み立てる

清書は「音声認識の誤変換だけを、会話の文脈から自然に直したもの」。以下を守る:

- **行の対応を1対1に保つ**: 原文が N 行なら清書も N 行。時刻・話者ラベル(`[時刻] 話者:` の頭)は原文と同一。本文だけを直す。
- **話し言葉のまま残す**: 「えーと」「あの」「〜だよね」など口語表現・言い淀みは削らない。要約や書き起こしの整形ではなく、あくまで誤変換の修正。
- **意味を変えない**: 元発話の意図を推測で書き換えない。特に自信が持てない箇所は原文を残す。
- **明白な誤変換だけ直す**: 「講座 → 口座」「機会 → 機械」「保証 → 保障」のような、文脈から強い確度で分かる置換に絞る。1音違いの候補が複数ある曖昧なケースは触らない。
- **話題に依存する固有名詞・専門用語**: ユーザーが会話の話題(「これは SF6 の話」「これは Kubernetes の運用会議」等)を伝えている場合、その領域の用語を優先して当てる。伝えられていない場合はユーザーに 1 度だけ話題を確認する。
- **原文で誤変換の可能性が低い箇所は変えない**: 意味は通っているが表記の好み(「よかったね」→「良かったね」など)を統一する等の整形はしない。

### 4. CLI で書き戻す

清書済みテキストを標準入力から `hearcat write-cleaned` に流す。

```sh
# 対象セッションが直近(latest)なら引数は不要。
cat <<'EOF' | hearcat write-cleaned
[10:00:01] 自分: 昨日銀行で口座を開設してきたよ。
...
EOF
```

過去のセッションを指定する場合は、セッション ID (ディレクトリ名) または セッションディレクトリのフルパスを引数に渡す。

```sh
cat <<'EOF' | hearcat write-cleaned 2026-07-08_024527
...
EOF
```

CLI は `cleaned.md` に書いたパスを標準出力に返す。ユーザーへの返答では、そのパスと「〇〇 箇所直しました」の要約を伝える。

### 5. 変更点を報告する

清書全文をユーザーへ流すのではなく、以下の要領で報告する:

- 対象セッション名(または時刻)
- 直した箇所の件数と、代表的な置換の例 3〜5 個(例: `講座→口座`, `機会→機械`)
- 全文が読みたい場合は `cleaned.md` を開くよう案内(パスを添える)。

## 何もしない/やめる場面

- 原文が空、または `hearcat latest` で該当が無い場合は、「文字起こしがまだありません」と伝えて終了。書き込みはしない。
- 明白な誤変換が 1 つも見つからなかった場合は、書き戻しをせずに「直す必要のある誤変換は見つかりませんでした」と伝えて終了。**空や無変更の `cleaned.md` を書かない**。
- ユーザーが「原文を直して」「上書きして」と言っても、原文には触れない。清書は必ず `cleaned.md` に保存する旨を伝える。

## 参考: HearCat の全体像

- 録音・文字起こし・セッション操作は `hearcat` スキル(親スキル)を参照。
- 保存ディレクトリ: `~/Library/Application Support/HearCat/sessions/`
- 原文ファイル名の規則: `<セッションディレクトリ名>.md`(固定名の `transcript.md` ではない)
- このスキルが生成する清書ファイル名: `cleaned.md`(セッションディレクトリの直下)
